# Workflow: Epic Creation (L3)

## Phase: planning

@pm gathering requirements and defining epic scope.

- Gather requirements from stakeholders and existing documentation
- Define epic scope, objectives, and success criteria

## Phase: breakdown

@pm breaking epic into stories with @sm.

- Break epic into stories using wave-based sequencing
- Each story must have clear dependencies and acceptance criteria

## Phase: tech_review

@architect reviewing technical feasibility.

- Assess complexity across 5 dimensions: scope, integration, infrastructure, knowledge, risk
- Validate architecture decisions and technology choices
- Identify technical risks and propose mitigations
